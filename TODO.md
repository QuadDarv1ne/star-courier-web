## 📝 Заметки

### Решения архитектуры
- Использовать Event Sourcing для игровых событий
- CQRS для разделения чтения/записи
- Микросервисы для масштабируемости (в будущем)

### Технологии на рассмотрении
- **Backend**: FastAPI → Litestar (для производительности)
- **Frontend**: Vue 3 → Nuxt 3 (для SSR)
- **Database**: SQLite → PostgreSQL (для продакшена)
- **Cache**: Memory → Redis (для распределённого кэша)

### Обратная связь от игроков
- Добавить больше романтических опций
- Улучшить баланс сложности
- Добавить интерактивную карту
- Увеличить количество сохранений
