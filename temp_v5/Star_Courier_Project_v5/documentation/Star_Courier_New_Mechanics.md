# Star Courier: Новые игровые механики

## Обзор

Данный документ описывает новые игровые механики, добавленные в главы 11-18 для расширения игрового опыта и подготовки к финальному противостоянию.

---

## 1. Система Резонанса (Resonance System)

### Описание
Резонанс — это способность Курьера чувствовать и взаимодействовать с аномалиями пространства и присутствием Сущности. Система развивается по мере прохождения игры и становится критически важной в финальных главах.

### Уровни Резонанса

| Уровень | Название | Разблокировка | Эффекты |
|---------|----------|---------------|---------|
| 1 | Основы | Глава 6 | Обнаружение аномалий в радиусе 100 единиц |
| 2 | Усиление | Глава 10 | Расширенный радиус (500), восприятие слабых мест Сущности |
| 3 | Мастерство | Глава 14 | Навигация через аномалии, коммуникация с Сущностью |
| 4 | Трансцендентный | Psychic 90+ | Манипуляция реальностью, разблокировка финала Слияния |

### Интеграция с существующими системами
- **Psychic способности**: Резонанс усиливает все Psychic-эффекты на 10-30%
- **Навигация**: Позволяет находить скрытые пути через аномальные зоны
- **Бой**: Предупреждение о приближении порождений Тьмы

### Код интеграции
```python
# Добавить в character stats
character["resonance_level"] = 1  # 1-4
character["resonance_experience"] = 0

# Проверка для разблокировки
def check_resonance_unlock(character):
    if character.get("psychic", 0) >= 90 and character.get("resonance_level") >= 3:
        character["resonance_level"] = 4
        character["merge_ending_available"] = True
```

## 2. Система Перевозки (Carrier System)

### Описание
Система перевозки управляет способностью корабля «Странник» транспортировать различные грузы, включая аномальные артефакты и образцы Сущности.

### Типы грузов

| Тип | Описание | Требования |
|-----|----------|------------|
| Стандартный | Обычные товары и ресурсы | Нет |
| Контрабанда | Скрытые отсеки | Veronica relationship 40+ |
| Аномальные артефакты | Артефакты из зон поражения | Anomaly Containment upgrade |
| Образцы Сущности | Останки порождений Тьмы | Entity Sample Storage |

### Улучшения корабля

```
cargo_optimization     → +50% грузоподъёмности
smuggler_compartments  → +20 скрытых слотов, 60% уклонение от сканирования
anomaly_containment    → Безопасный транспорт аномалий
entity_sample_storage  → Доступ к ингредиентам для Alchemy
```

### Интеграция с экономикой
- Контрабанда даёт +50% прибыли, но риск конфискации
- Аномальные артефакты — уникальные товары для исследователей
- Образцы Сущности — ингредиенты для продвинутых Alchemy рецептов

## 3. Система Выбора Пути (Path System)

### Описание
В главе 13 игрок выбирает одну из трёх фракций для поддержки в борьбе с Сущностью. Выбор определяет доступные ресурсы, союзников и вариации финала.

### Три Пути

#### Путь Альянса
```
Преимущества:
- Доступ к флоту Объединённых Миров
- Официальное признание и награды
- Военная поддержка в финальной битве
- Финансовые ресурсы (+5000 кредитов)

Требования:
- Выполнение миссий для Альянса
- Положительная репутация с официальными структурами

Финальная вариация:
- Флот прикрывает эвакуацию пограничных миров
- Курьер получает статус героя Альянса
```

#### Путь Наблюдателя
```
Преимущества:
- Доступ к древним знаниям Ордена
- Psychic усиление (+20)
- Обучение Резонансу
- Поддержка рыцарей Ордена в финале

Требования:
- Высокий уровень Psychic (40+)
- Уважение Зары

Финальная вариация:
- Зара основывает школу изучения аномалий
- Курьер становится живым архивом знаний
```

#### Путь Независимости
```
Преимущества:
- Свобода действий
- Доступ к сети независимых агентов
- Контрабандистские маршруты
- Поддержка наёмников в финале

Требования:
- Отношения с Волковым 30+
- Независимая репутация

Финальная вариация:
- Создание Конфедерации Независимых Миров
- Курьер — легенда среди свободных капитанов
```

### Реализация
```python
PATH_RESOURCES = {
    "alliance": {
        "credits": 5000,
        "fleet_support": True,
        "equipment_discount": 40
    },
    "observer": {
        "psychic_boost": 20,
        "ancient_knowledge": True,
        "resonance_boost": 2
    },
    "independence": {
        "network_access": True,
        "smuggler_routes": True,
        "mercenary_discount": 50
    }
}

def apply_path_bonus(character, path):
    for key, value in PATH_RESOURCES[path].items():
        character[key] = value
    character["path"] = path
```

## 4. Система Финального Выбора (Ending System)

### Описание
В главе 18 игрок делает финальный выбор, определяющий концовку игры. Выбор зависит от накопленных решений, развития способностей и отношений.

### Три Финала

#### Изгнание (Exile)
```
Условия:
- Любой путь
- Решимость против Сущности

Механика:
- Уничтожение якоря Сущности
- Требуется жертва (Курьер или соратник)
- Высокий уровень отношений позволяет разделить ношу

Последствия:
- Сущность изгнана на тысячелетия
- Галактика спасена
- Курьер связан с местом победы
```

#### Договор (Treaty)
```
Условия:
- Psychic 70+ ИЛИ Empathy 80+
- Понимание природы Сущности

Механика:
- Перенастройка станции
- Заключение долгосрочного соглашения
- Курьер становится Хранителем Границы

Последствия:
- Ограниченное сосуществование
- Часть галактики — запретная зона
- Время для развития цивилизаций
```

#### Слияние (Merge)
```
Условия:
- Psychic 90+
- Resonance Level 4
- Entity Communion способность

Механика:
- Интеграция Сущности в сознание Курьера
- Трансцендентная эволюция
- Контроль над аппетитами Сущности

Последствия:
- Курьер становится новой силой галактики
- Управляемая эволюция вместо хаотичного поглощения
- Потеря части человечности
```

### Романтические концовки
При высоком уровне отношений (80+) с романтическим партнёром, финал включает уникальную сцену:
- Партнёр может разделить судьбу Курьера
- Эпилог показывает совместное будущее
- Уникальные диалоги в финальных сценах

## 5. Система Ментального Состояния (Mental State)

### Описание
В главах 16-17 добавляется отслеживание ментального состояния Курьера и команды под воздействием Сущности.

### Параметры
```
mental_health: 0-100 (100 = идеально)
entity_influence: 0-100 (0 = чист)
corruption_risk: 0-100 (вероятность падения)
```

### Влияющие факторы
- Psychic способности → защита от влияния
- Отношения с командой → моральная поддержка
- Illicit substances → временное ослабление, долгосрочный риск
- Контакт с Сущностью → рост influence

### Пороги
| Mental Health | Эффект |
|---------------|--------|
| 80-100 | Нормальное состояние |
| 60-79 | Лёгкие галлюцинации |
| 40-59 | Видения, снижение навыков |
| 20-39 | Серьёзные нарушения, риск выбора Слияния |
| 0-19 | Потеря контроля, автоматический финал Слияния |

## 6. Интеграция с существующим кодом

### Изменения в characters.py
```python
# Добавить новые поля
CHARACTER_TEMPLATE = {
    # ... существующие поля ...
    "resonance_level": 1,
    "mental_health": 100,
    "entity_influence": 0,
    "path": None,  # "alliance", "observer", "independence"
    "ending_available": ["exile"]  # Разблокированные концовки
}
```

### Изменения в quests.py
```python
# Добавить проверку пути
def get_available_quests(character, chapter):
    path = character.get("path")
    quests = []
    
    for quest in ALL_QUESTS:
        # Проверка пути
        if quest.get("path_required") and quest["path_required"] != path:
            continue
        
        # Проверка главы
        if quest.get("chapter") != chapter:
            continue
            
        quests.append(quest)
    
    return quests
```

### Изменения в dialogues.py
```python
# Добавить проверки для финального выбора
def get_dialogue(dialogue_id, character):
    dialogue = DIALOGUES.get(dialogue_id)
    
    # Проверка требований для финальных диалогов
    if dialogue.get("ending_requirement"):
        if dialogue["ending_requirement"] not in character.get("ending_available", []):
            return None  # Диалог недоступен
    
    return dialogue
```

## Заключение

Новые механики добавляют глубину финальной части игры:
- **Резонанс** — связь с основным антагонистом
- **Carrier** — подготовка к битве через ресурсы
- **Path** — политический выбор с последствиями
- **Ending** — персонализированный финал
- **Mental State** — психологическое давление

Все механики интегрируются с существующими системами и не требуют полной переработки кодовой базы.
